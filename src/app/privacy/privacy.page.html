Privacy & Security Overview This mobile inventory system is designed to manage product records such as item name, category, quantity, price, supplier name, stock status, featured item, and special notes. Because the app connects to a remote database through RESTful APIs, privacy and security are important to protect business data and keep records accurate. Data Security The application sends and receives inventory records through a remote server instead of storing all data permanently on the device. This reduces the risk of users directly viewing database credentials. The app also uses structured API requests for adding, updating, searching, and deleting items, which helps keep communication organised and consistent. User Privacy This application focuses on inventory management only. It does not ask for unnecessary personal data such as user location, contact lists, photos, or microphone access. The main data handled by the app relates to stock records rather than sensitive personal information, which helps reduce privacy risk. Input Validation Validation is important in this inventory app because incorrect values could damage the database. For example, quantity and price should be numeric, item name is required, supplier name is required, and some fields use controlled options such as category and stock status. This reduces invalid input and improves data integrity. Current Risks and Limitations Although the app includes validation and structured API calls, there are still limitations. For example, the app does not currently include user login, role-based access control, or detailed audit tracking. This means that, in a real business environment, stronger controls would still be needed to prevent unauthorised updates or deletions. Future Improvements Future improvements for this inventory app could include secure login, admin and staff permission levels, stronger server-side validation, encrypted authentication tokens, and activity logs for tracking who added, updated, or deleted an item. These improvements would make the system more suitable for real commercial use.
